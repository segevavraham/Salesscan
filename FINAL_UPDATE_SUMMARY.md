## 🎯 מערכת Priority Keys

המערכת בוחרת מפתחות לפי סדר עדיפות:

```
Priority 1: Master API Keys (מה שהגדרת באדמין)
    ↓
Priority 2: User-specific keys (אם האדמין הקצה למשתמש ספציפי)
    ↓
Priority 3: Personal keys (מהאופציות - תאימות לאחור)
```

**משמעות:**
- אם הגדרת מפתח באדמין → כל המשתמשים ישתמשו בו
- אם לא → המערכת תחפש במקומות אחרים
- שמירת עלויות מרכזית!

### כשאתה לוחץ "Start":
```
1. המערכת טוענת config מ-Chrome storage

6. מזהה דוברים (מוכר/לקוח)
7. כשהלקוח מדבר → שולחת ל-OpenAI
8. מקבלת טיפים ומציגה אותם
9. שומרת היסטוריה ל-storage
```

### זרימת מפתחות:
```
loadConfig() → בדוק Master Keys →
   יש? → השתמש בהם
   אין? → בדוק User Keys →
      יש? → השתמש בהם
      אין? → בדוק Personal Keys →
         יש? → השתמש בהם
         אין? → הצג אזהרה
```
